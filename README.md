# Studia AI

ქართული AI სასწავლო სივრცე უნივერსიტეტის სტუდენტებისთვის. პროექტი აერთიანებს AI ტუტორს, სასწავლო გეგმას, ფაილებზე დაფუძნებულ კითხვებსა და ინტერაქტიულ ანალიტიკას.

## შესაძლებლობები

- ქართული ინტერფეისი და ქართულენოვანი AI პასუხები;
- ოთხი რეჟიმი: ახსნა, სოკრატული ტუტორი, ტესტი და კონსპექტი;
- PDF, DOCX, PPTX, TXT, MD და CSV მასალის რეალური მიმაგრება და კონტექსტად გამოყენება;
- Google ანგარიშით შესვლა და თითოეული დადასტურებული ელფოსტისთვის სრულად განცალკევებული მონაცემები;
- გამოცდების ავტომატური დამატება, განახლება და წაშლა დაკავშირებულ Google Calendar-ში;
- სასწავლო დავალებებისა და გამოცდების ანგარიშზე შენახვა;
- მასალების R2 საცავში შენახვა და ნებისმიერი მოწყობილობიდან მიღება;
- საიტის ხილულ ტაბში გატარებული სწავლის დროის ავტომატური ცოცხალი ტაიმერი;
- პროგრესის Chart-ები, heatmap და გასაუმჯობესებელი თემები;
- სტუდენტის პროფილი და პასუხის სტილის პარამეტრები;
- Responsive და keyboard-friendly ინტერფეისი;
- დემო რეჟიმი API გასაღების გარეშე;
- უსაფრთხო server-side AI proxy — გასაღები ბრაუზერში არ ხვდება.

## ტექნიკური სტრუქტურა

პროექტს გარე ბიბლიოთეკები და `npm install` არ სჭირდება. აპლიკაცია მუშაობს ერთ Worker-ში, ხოლო ანგარიშის მონაცემებს D1 და R2 ინახავს:

```text
worker/index.js        აპლიკაცია და API proxy
.openai/hosting.json  ChatGPT Sites-ის კონფიგურაცია
drizzle/               D1 მონაცემთა ბაზის მიგრაცია
scripts/               build, validation და იზოლაციის ტესტები
.github/workflows/    GitHub CI შემოწმება
```

## AI-ის ჩართვა

ჰოსტინგის Environment Variables/Secrets განყოფილებაში დაამატე:

```text
GEMINI_API_KEY=შენი_api_გასაღები
GEMINI_MODEL=gemini-3.5-flash
GOOGLE_CLIENT_ID=შენი_web_client_id.apps.googleusercontent.com
GOOGLE_OAUTH_CLIENT_SECRET=შენი_oauth_client_secret
CALENDAR_TOKEN_KEY=32_ბაიტიანი_base64url_გასაღები
```

`GEMINI_MODEL` არასავალდებულოა. გასაღების გარეშე საიტი ავტომატურად იყენებს უსაფრთხო დემო პასუხებს. უფასო Gemini API კვოტა ხელმისაწვდომობაზე, მოდელსა და რეგიონზეა დამოკიდებული; ლიმიტის ამოწურვისას აპლიკაცია აჩვენებს გასაგებ შეტყობინებას და დემო პასუხზე გადადის.

გასაღები არასდროს ჩაწერო `worker/index.js`-ში, Git commit-ში ან საჯარო GitHub Secret-ის მნიშვნელობად.

## შემოწმება

```bash
npm test
npm run build
npm run validate
```

ეს ბრძანებები პაკეტების ინსტალაციას არ საჭიროებს.

## GitHub-ზე ატვირთვა

1. შექმენი ცარიელი GitHub repository.
2. ატვირთე პროექტის ყველა ფაილი.
3. `main` branch-ზე ყოველი push ავტომატურად გაუშვებს სინტაქსის, ანგარიშების იზოლაციისა და build artifact-ის შემოწმებას.
4. საიტი გამოაქვეყნე ChatGPT Sites-ით — ის ავტომატურად ამატებს `.openai/hosting.json`-ში აღწერილ D1/R2 რესურსებს. Google Cloud-ში production URL დაამატე Authorized JavaScript Origins-ში.

Google Sign-In-ის ID token სერვერზე მოწმდება Google-ის საჯარო გასაღებით; კლიენტის მიერ ცალკე გამოგზავნილი ელფოსტის ნდობა დაუშვებელია. Calendar API-ის გამოყენებამდე სტუდენტი ცალკე აძლევს `calendar.events` ნებართვას. refresh token D1-ში AES-GCM-ით დაშიფრული ინახება.

## მონაცემები და უსაფრთხოება

- Google-ის დადასტურებული ელფოსტა გამოიყენება მხოლოდ ანგარიშის საიდენტიფიკაციოდ; Google პაროლი აპლიკაციას არ გადაეცემა და Gmail/Drive უფლებები არ მოითხოვება.
- პროფილი, დავალებები, გამოცდები და სწავლის დრო ინახება D1-ში ელფოსტის მიხედვით; ფაილები — R2-ში ანგარიშის ჰეშირებული გასაღებით.
- `localStorage` და `IndexedDB` მხოლოდ არჩევითი ოფლაინ ქეშია. ანგარიშის შეცვლისას სხვა მომხმარებლის ადგილობრივი ქეში იშლება.
- ფაილის შინაარსი იგზავნება AI-სთან მხოლოდ მომხმარებლის კონკრეტული მოთხოვნისას.
- DOCX და PPTX ტექსტი ამოიღება Worker-ში გარე ბიბლიოთეკების გარეშე; PDF და CSV Gemini-ს პირდაპირ დოკუმენტად გადაეცემა.
- ფაილის მაქსიმალური ზომაა 8 MB.
- სწავლის ტაიმერი ითვლის მხოლოდ ანგარიშში შესვლის შემდეგ ხილულ ტაბში გატარებულ დროს და პერიოდულად სერვერზე სინქრონდება.
- API ამოწმებს მოთხოვნის ზომას, ტექსტის სიგრძეს და დასაშვებ რეჟიმებს.
- ყველა პირადი API მოითხოვს მოქმედ, `HttpOnly` Google სესიას და მონაცემთა მოთხოვნებს დადასტურებული ელფოსტით ზღუდავს.
